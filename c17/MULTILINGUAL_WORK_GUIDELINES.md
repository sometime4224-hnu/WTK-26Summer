# c17 다국어 기능 작업 지침

작성일: 2026-06-05

## 목표

c17의 다국어 기능은 한국어 학습을 대체하는 전체 번역 기능이 아니라, 학생이 한국어 과제를 수행할 수 있도록 돕는 비계 기능이다. 한국어 본문, 문법 형태, 정답, 학생이 직접 산출해야 하는 표현은 가능한 한 한국어로 유지하고, 활동 지시와 개념 설명을 필요한 만큼만 보조한다.

## 지원 언어

- 기본 언어: 한국어 `ko`
- 추가 언어: 영어 `en`, 베트남어 `vi`, 몽골어 `mn`, 아랍어 `ar`, 카자흐어 `kk`, 태국어 `th`

언어 작업은 반드시 한 언어씩 완료한다. 여러 언어의 번역 데이터를 동시에 추가하거나 검증하지 않는다.

## 이번 작업의 포함 범위

### 1차 필수 대상: 문법 본 페이지

- `c17/grammar1.html`
- `c17/grammar2.html`
- `c17/grammar3.html`
- `c17/grammar4.html`
- `c17/grammar-main.js`
- `shared/multilang-toggle.js`
- `shared/multilang-toggle.css`

문법 페이지에서는 다음 요소에 다국어 비계를 제공한다.

- 문법 의미 설명
- 형태 결합 설명
- 비슷한 문법과의 차이
- 사용 맥락과 주의점
- 활동 지시와 짧은 도움말

다음 요소는 번역하지 않거나 한국어 표기를 보호한다.

- 문법 표지: 예, `-고 보니`, `A-(으)ㄴ/V-는 척하다`, `-다니요`, `N(이)라고 해서`
- 조사, 어미, 활용 형태
- 학생이 답해야 하는 한국어 문장
- 한국어 예문 안의 핵심 학습 표현

### 2차 검토 대상: 읽기 페이지

- `c17/reading.html`
- `c17/reading-cuttoon.html`

읽기 페이지는 문법 페이지 완료 후 별도 검토한다. 본문 전체 번역은 기본 방향이 아니며, 활동 지시, 어려운 표현, 질문 이해 보조를 중심으로 적용한다.

### 조건부 검토 대상: 문법 보조 자료

- `c17/grammar1-support-quiz.html`
- `c17/grammar2-support-quiz.html`
- `c17/grammar3-support-material1.html`
- `c17/grammar4-support-material1.html`

문법 본 페이지 4개와 읽기 페이지의 기본 구조가 안정된 뒤에만 포함 여부를 결정한다. 포함하더라도 언어별 순차 작업 원칙을 그대로 따른다.

## 이번 작업의 제외 범위

### 듣기 페이지

- `c17/listening1.html`
- `c17/listening2.html`
- `c17/listening-data.js`
- `shared/listening-workbook.js`
- `shared/listening-cuttoon-sync.js`

듣기 페이지는 이번 다국어 데이터 생성 대상에서 제외한다. 듣기는 번역 기능을 비활성화한 상태로 유지하며, 학생 화면에 번역 버튼이나 언어 전환 버튼을 제공하지 않는다. 기존에 남아 있는 다국어 데이터가 있더라도 이번 작업에서 확장하거나 새 언어 데이터를 만들지 않는다.

### 어휘 페이지

- `c17/vocabulary.html`
- `c17/vocab-support-*.html`

어휘는 이미지, 예문, 뜻풀이, 분류 체계를 함께 재설계해야 하므로 이번 작업에서 제외한다.

### 쓰기와 말하기 페이지

- `c17/writing-cut.html`
- `c17/writing-cut-teacher.html`
- `c17/grammar*-speaking-pro.html`

쓰기와 말하기는 학생 산출 활동이 중심이므로 이번 작업에서 제외한다. 추후 다룰 경우에도 활동 지시 번역부터 낮은 강도로 시작한다.

## 공통 구현 원칙

1. 번역 블록은 `data-lang="{code}"`와 `lang-box` 패턴을 우선 사용한다.
2. 언어 선택 UI는 `shared/multilang-toggle.js`와 `shared/multilang-toggle.css`를 공통으로 사용한다.
3. 페이지별 언어 목록에는 `["en", "vi", "mn", "ar", "kk", "th"]` 순서를 사용한다.
4. 이전 베트남어 전용 토글이나 `viToggle` 방식은 새 작업에서 확장하지 않는다.
5. 한국어 문법 표지는 `<code>`로 감싸고, 아랍어 블록 안에서는 `class="ko-term" lang="ko" dir="ltr"`를 추가한다.
6. 아랍어 번역 블록은 오른쪽에서 왼쪽으로 읽히도록 `dir="rtl"` 규칙을 유지한다.
7. 아랍어 문장 안의 한국어 문법 표지, 숫자, 파일명, 코드형 텍스트가 뒤집히지 않는지 반드시 눈으로 확인한다.
8. 번역은 학습자의 이해를 돕는 짧은 설명으로 작성한다. 원문보다 길어져 화면을 밀어내는 번역은 줄인다.
9. 버튼, 카드, 표 안의 번역 텍스트가 모바일 폭에서 넘치지 않는지 확인한다.
10. 한 언어 작업 중 다른 언어의 문구를 함께 고치지 않는다. 단, 공통 렌더러 버그는 별도 변경으로 명확히 기록한다.

## 언어별 작업 완료 기준

한 언어는 아래 조건을 모두 만족해야 완료로 본다.

- 대상 페이지의 해당 언어 번역 블록이 모두 존재한다.
- 언어 토글에서 해당 언어 버튼이 보인다.
- 해당 언어를 켰을 때 번역 블록만 나타나고 다른 언어 블록은 숨겨진다.
- 번역을 끄면 한국어 기본 화면만 남는다.
- 한국어 문법 표지와 핵심 표현이 번역되지 않고 보존된다.
- 아랍어 작업일 경우 RTL과 한국어 표지 방향이 모두 정상이다.
- 데스크톱과 모바일 폭에서 텍스트 겹침, 카드 찌그러짐, 버튼 넘침이 없다.
- `node --check` 대상 스크립트가 통과한다.
- 브라우저에서 `grammar1.html`부터 마지막 대상 페이지까지 직접 확인한다.

## 구체적 작업 순서

### 0단계: 기준 구조 정리

1. `grammar1.html`의 현재 다국어 구조를 기준 파일럿으로 확인한다.
2. `shared/multilang-toggle.js`의 언어 목록에 필요한 언어가 모두 있는지 확인한다.
3. `shared/multilang-toggle.css`에 `en`, `vi`, `mn`, `ar`, `kk`, `th` 스타일이 모두 있는지 확인한다.
4. `grammar-main.js`에 남은 베트남어 전용 토글 의존성이 실제 화면에 필요한지 정리한다.
5. 듣기, 어휘, 쓰기, 말하기 파일은 이번 단계에서 수정하지 않는다.

### 1단계: 영어 `en`

1. `grammar1.html`에 영어 번역 블록을 추가하고 언어 목록에 `en`을 포함한다.
2. 영어 화면에서 공통 토글, 카드 높이, 모바일 줄바꿈을 검증한다.
3. 같은 패턴으로 `grammar2.html`, `grammar3.html`, `grammar4.html`에 영어를 추가한다.
4. 문법 표지와 예문 핵심 표현이 영어로 바뀌지 않았는지 확인한다.
5. 네 문법 페이지를 브라우저에서 모두 확인하고, 영어 작업 완료를 기록한다.

### 2단계: 베트남어 `vi`

1. `grammar1.html`의 기존 베트남어 블록을 새 언어 순서와 공통 규칙에 맞춰 정리한다.
2. `grammar2.html`, `grammar3.html`, `grammar4.html`에 베트남어 번역 블록을 추가한다.
3. 기존 베트남어 전용 UI가 새 공통 토글과 충돌하지 않는지 확인한다.
4. 네 문법 페이지를 브라우저에서 모두 확인하고, 베트남어 작업 완료를 기록한다.

### 3단계: 아랍어 `ar`

1. `grammar1.html`의 기존 아랍어 블록을 RTL 기준으로 재검토한다.
2. 모든 한국어 문법 표지에 `class="ko-term" lang="ko" dir="ltr"`가 필요한지 확인한다.
3. `grammar2.html`, `grammar3.html`, `grammar4.html`에 아랍어 번역 블록을 추가한다.
4. `-고 보니`처럼 앞의 하이픈이 뒤로 가는 문제가 없는지 각 페이지에서 확인한다.
5. 네 문법 페이지를 브라우저에서 모두 확인하고, 아랍어 작업 완료를 기록한다.

### 4단계: 몽골어 `mn`

1. `grammar1.html`의 몽골어 블록을 공통 표현 톤에 맞춰 재검토한다.
2. `grammar2.html`, `grammar3.html`, `grammar4.html`에 몽골어 번역 블록을 추가한다.
3. 긴 문장이 카드와 버튼 영역을 밀어내지 않는지 확인한다.
4. 네 문법 페이지를 브라우저에서 모두 확인하고, 몽골어 작업 완료를 기록한다.

### 5단계: 카자흐어 `kk`

1. `grammar1.html`의 카자흐어 블록을 공통 표현 톤에 맞춰 재검토한다.
2. `grammar2.html`, `grammar3.html`, `grammar4.html`에 카자흐어 번역 블록을 추가한다.
3. 키릴 문자와 한국어 문법 표지가 같은 줄에서 어색하게 붙지 않는지 확인한다.
4. 네 문법 페이지를 브라우저에서 모두 확인하고, 카자흐어 작업 완료를 기록한다.

### 6단계: 태국어 `th`

1. `grammar1.html`의 태국어 블록을 공통 표현 톤에 맞춰 재검토한다.
2. `grammar2.html`, `grammar3.html`, `grammar4.html`에 태국어 번역 블록을 추가한다.
3. 태국어 줄바꿈이 카드 폭을 넘기지 않는지 확인한다.
4. 네 문법 페이지를 브라우저에서 모두 확인하고, 태국어 작업 완료를 기록한다.

### 7단계: 읽기 페이지 적용 여부 검토

1. 문법 4개 페이지의 6개 추가 언어가 모두 완료된 뒤에만 시작한다.
2. `reading.html`과 `reading-cuttoon.html`에서 번역이 필요한 영역과 제외할 영역을 먼저 표시한다.
3. 본문 전체 번역을 넣지 않는다는 원칙을 다시 확인한다.
4. 읽기 페이지도 `en -> vi -> ar -> mn -> kk -> th` 순서로 한 언어씩 진행한다.

## 검증 명령

문법 페이지 작업 중 스크립트를 수정했다면 최소한 아래 검사를 실행한다.

```powershell
node --check c17/grammar-main.js
node --check shared/multilang-toggle.js
```

듣기 제외 정책을 건드리지 않았는지 확인할 때는 브라우저에서 `listening1.html`, `listening2.html`에 번역 버튼이 보이지 않는지만 확인한다. 듣기 데이터 번역을 새로 만들지 않는다.

## 변경 기록 방식

각 언어 작업이 끝날 때 작업 메모에 다음을 남긴다.

- 완료 언어
- 완료 페이지
- 수정 파일
- 브라우저 확인 결과
- 남은 문제
- 다음 작업 언어
