# 소담한 마을 — 정식 그래픽 개선판

2026-07-11 그래픽 개선 사본을 `c14/farming-game` 정식 버전으로 승격했습니다. 이전 개선판 `c14/farming-game-improved`는 비교와 복구를 위해 그대로 보존합니다.

시골 동네를 돌아다니며 표현을 몸으로 익히는 브라우저 게임입니다. 퀴즈형 교육 게임처럼 보이지 않도록, 플레이어는 주말농장에 내려온 인물이 되어 마을의 일을 도우며 표현을 장면과 행동으로 기억하게 됩니다.

## 이번 그래픽 개선

- 정원·잔디밭·채소밭·큰 밭·축사·연못 주변의 큰 단색 사각 구역을 유기적인 지형 경계와 2단 명암으로 교체
- 흙길에 부드러운 길어깨, 둥근 교차부, 바퀴 자국, 자갈, 길가 풀을 추가하고 화면 밖 장식은 그리지 않도록 정리
- 사진풍 월드 나무·작물·가축 이미지를 절차형 동화책 일러스트로 통일하고 수관·줄기·접지 그림자를 보강
- 아침·낮·저녁·밤에 따라 그림자 방향과 길이, 지면 색조, 해·달 원반, 창문·등불의 밝기가 함께 변하도록 개선
- 연못에 시간대별 물색, 잔물결, 물빛 반사, 밤 달빛 띠, 습지 가장자리를 추가
- 활동 장면의 오두막·밭·축사·연못 원경이 지면에 가려지던 그리기 순서를 수정
- 정원·잔디 활동의 작업 지형과 덤불을 유기적인 형태와 명암으로 개선하고 스마트폰에서 현재 대상을 더 굵게 강조
- 일곱 활동의 논리 화면을 640px 폭으로 통일해 스마트폰 세로에서도 오른쪽 작업 대상이 잘리지 않도록 조정
- 모종·성장 채소·소·돼지·양·물고기를 같은 동화책풍 절차형 그림으로 통일하고, 물·모종·사료·바구니 수량을 실제 진행 상태와 맞춤
- 들고 있는 도구가 바닥에도 남던 중복을 없애고, 잔디의 전후 변화·흙고랑·곡물 사료·자연형 연못을 행동 결과가 잘 보이도록 보강
- 스마트폰 활동 화면의 하단 안내막과 완료 카드를 축소해 플레이어·도구·마지막 변화 장면이 가려지지 않도록 개선
- 스마트폰에서는 비목표 표식과 중복 방향 표식을 줄이고, 저사양 프로필은 30fps·고사양 스마트폰은 45fps로 제한해 안정성 확보
- 이미지·음성을 실제 사용 시점에 불러오고, 화면 밖 건물·나무·지형·광원은 그리지 않으며, 미니맵과 반복 DOM 갱신 주기를 줄여 초기 로딩과 프레임 부담 완화
- 시간 표시를 `06시 아침 → 11시 점심 → 15시 낮 → 18시 저녁 → 21시 밤` 흐름에 맞춰 보정
- 정식 저장 키를 사용하고 기존 개선판·그래픽 검수판·이전 정식판의 진행 기록을 자동 이전

자동 검증은 `tests/e2e/c14-farming-game-graphics.spec.js`에 있으며 320×568, 390×844, 430×932, 480×320, 568×320, 667×375, 844×390, 768×1024, 1024×768 화면과 일곱 활동 장면, 저장 이전, 야간 색감, 음성 지연 로딩을 확인합니다.

## 개선 내용

- 스마트폰 세로, 태블릿 세로·가로에서 게임과 조작부를 한 화면 안에 배치
- 시작 카드의 권장 경로를 `전체화면 → 첫 방문 조작 안내 → 시작/이어하기`로 연결하고, 전체화면 미지원·거부 시 일반 화면으로 계속 진행
- 스마트폰 전체화면은 `세로모드` 잠금을 먼저 시도하고, 잠금 미지원·거부 시 기기를 세로로 돌릴 때까지 회전 안내를 표시
- 첫 터치 방문에서 실제 왼쪽 이동 패드를 밀고 오른쪽 행동 버튼을 눌러 보는 2단계 안내를 제공하며, 안내 패널에서 다시 볼 수 있도록 구성
- 스마트폰 가로에서도 캔버스가 위로 잘리지 않고 중앙 안내와 양쪽 엄지 조작부가 겹치지 않도록 전용 레이아웃 추가
- 상단 이동바, 현재 부탁, 안내 버튼, 조이스틱의 겹침 해소
- 첫 화면을 `이모에게 가기`라는 다음 학습 행동 중심으로 간소화
- 터치 화면에서 행동 안내를 `행동 버튼 · …`으로 표시
- 진행 중인 작업·대화·표현 퀴즈를 자동 저장하고 복원하도록 보강
- 미니게임에 `저장하고 나가기` 추가
- 대화를 끝까지 들은 후에만 듣기 표현과 스토리 진행을 완료하도록 수정
- 표현 퀴즈의 정답 선노출 제거, 엔딩에 표현 확인 결과와 놓친 표현 요약 추가
- 초기화·새로 시작을 두 번 눌러야 하도록 해 저장 기록 실수 방지
- 학습자 화면의 영어 보조 문구를 한국어로 통일하고 상태 알림·대화창 접근성 보강
- 움직임 축소 설정에서 효과 밀도와 화면 전환 축소
- 마지막 손동작 뒤 바뀐 장면과 전체 표현을 잠시 보여 주는 공통 완료 연출
- 전원 생활의 느낌 표현을 바람·고요·해의 이동·진흙 발자국으로 보여 주는 월드 장면
- 활동을 먼저 끝낸 뒤에만 해당 표현 팻말이 열리는 행동→회상 순서
- 아침·낮·저녁·밤의 마당 이모 화면 대사와 실제 음성 일치

## 기획 요약

- 장르: 짧은 탑다운 내러티브 어드벤처
- 톤: 조용하고 따뜻한 시골 하루
- 플레이 구조: 부탁을 따라 움직이는 챕터형 하루 진행
- 학습 방식: `정원`, `잔디`, `채소`, `농사`, `가축`, `물고기` 같은 대상과 `가꾸다`, `깎다`, `심다`, `키우다`, `짓다`, `잡다` 같은 행동을 실제 상호작용으로 연결
- 비교육 게임처럼 보이게 하는 장치:
  - 표현은 UI 카드보다 공간과 행동에서 먼저 만남
  - 몸으로 활동을 끝낸 뒤에만 작은 팻말 회상이 열림
  - 전원 생활 표현은 사건과 분위기에서 해금
  - 듣기 표현은 NPC 대사와 생성된 음성 파일로 노출

## 플레이 확장

- 상단 HUD에서 `현재 부탁`, `진행도`, `마을 온기`, `바구니`를 확인
- `미니맵`과 화면 밖 목표 화살표로 현재 부탁 위치를 따라가기 쉬움
- `자동 저장`과 `이어서 걷기`로 중간 진행, 대화 위치, 표현 선택, 진행 중인 작업을 다시 불러올 수 있음
- 각 일감은 별도 활동 장면으로 전환되며, 작은 캐릭터를 직접 움직여 도구를 집고 옮기고 사용하는 방식으로 진행
- 버스 정류장 → 마당 정리 → 밭일 → 오후 심부름 → 저녁 상차림 → 언덕 마무리로 이어지는 스토리 단계 추가
- `할 일판` 상호작용으로 다음 목표를 다시 확인 가능
- 일을 마칠수록 바구니가 채워지고 마을 온기가 올라감
- 오후가 되면 등불과 반딧불이가 나타나며 마을 풍경이 바뀜
- 이모가 마당에서 중간 진행 상황에 맞는 후속 대사를 들려줌
- 마지막에는 집 앞 식탁과 언덕을 거쳐 하루를 정리하는 엔딩 카드 표시

## 표현 설계

| 표현 | 게임 안 장면 |
| --- | --- |
| 정원을 가꾸다 | 작업대에서 전지가위를 집고, 덤불 앞으로 걸어가 세 곳을 직접 다듬기 |
| 잔디를 깎다 | 잔디깎는 기계를 붙잡고 긴 잔디 줄을 따라 직접 밀고 지나가기 |
| 채소를 심다 | 모종 상자를 들고 빈 흙 자리마다 옮겨 다니며 하나씩 심기 |
| 채소를 키우다 | 양동이를 들고 우물에서 물을 채운 뒤 시든 줄마다 직접 물 주기 |
| 농사를 짓다 | 호미를 들고 밭고랑 세 줄 위를 직접 걸으며 흙 다지기 |
| 가축을 키우다 | 사료 자루를 집어 외양간 안쪽 먹이통 두 곳까지 직접 옮겨 채우기 |
| 물고기를 잡다 | 뜰채로 물고기를 건진 뒤 바구니까지 옮겨 담기 |

## 전원 생활 표현 처리

- `공기가 맑다`: 마을 윗길과 연못 쪽으로 나가면 자연스럽게 해금
- `여유가 있다`: 벤치에서 쉬는 행동으로 체감
- `시간 가는 줄 모르다`: 여러 일을 마치면 오후로 바뀌며 해금
- `불편하다`: 비 온 뒤 진흙길을 지나며 느리게 움직이게 함
- `따분하다`: 동네 아이의 대사로 시골에 대한 다른 시선도 같이 제시
- `평화롭다`: 언덕에서 마을 전체를 내려다볼 때 마무리 정서로 제시

## 듣기 표현 처리

- `주말농장`
- `시골에 내려가다`
- `직접 채소를 키우다`
- `처음 농사를 짓다`

위 네 표현은 버스 정류장 이모의 두 줄 대사 안에서 자연스럽게 등장합니다. 대사는 생성된 음성 파일을 재생하는 방식으로 처리합니다.

## 음성 생성

대사는 브라우저 TTS가 아니라 미리 생성한 mp3 파일을 재생하도록 구성했습니다.

1. `py -3 scripts/generate_voice.py`
2. `py -3 scripts/generate_sfx.py`
3. 생성된 파일은 `audio/` 폴더에 저장됩니다.

기본 설정은 `edge-tts`의 한국어 신경망 음성을 사용하며, 이모와 아이가 서로 다른 음색으로 들리도록 분리해 두었습니다.

## 실행 방법

빌드 도구 없이 바로 실행할 수 있습니다.

1. `index.html`을 브라우저로 엽니다.
2. 또는 정적 서버로 열어도 됩니다. 예: `python -m http.server`

## 조작

- 이동: `WASD` 또는 방향키
- 상호작용: `E` 또는 `Space`
- 활동 장면: 같은 이동/행동 키로 도구를 집고 옮기고 사용
- 작업 저장 후 나가기: 화면의 `저장하고 나가기` 버튼 또는 `Esc`
- 모바일: 시작할 때 `세로 전체화면으로 시작`을 누른 뒤 화면 아래 왼쪽 이동 패드와 오른쪽 `행동` 버튼 사용

## 파일 구성

- `index.html`: 게임 화면과 UI 구조
- `styles.css`: 비주얼 테마와 반응형 레이아웃
- `game.js`: 월드 렌더링, 이동, 활동 장면, 표현 해금 로직
