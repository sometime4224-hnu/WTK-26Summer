# 14과 어휘 보조활동 제작 설계 근거 보고서

- 작성일: 2026-07-11
- 제작 페이지: `c14/vocab-support-lab.html`
- 페이지명: **도시와 변화 어휘 연구소**
- 저장 키: `c14-vocab-support-city-change-v1`

## 1. 제작 목적과 범위

14과 핵심 어휘 28개 가운데 기존 **농촌 마을 산책 게임**이 반복 학습하는 18개를 제외하고, 남은 10개만 새 보조활동의 정답 어휘로 사용했다.

### 1.1 범위 산정 근거

- 전체 28개: `c14/vocabulary.html`의 `window.VOCABULARY_CONFIG.words`
- 기존 게임의 명사·동작 어휘: `c14/farming-game/game.js`의 `nounWords`, `verbWords`, `collocations`
- 기존 게임의 생활 표현: 같은 파일의 `moodExpressions`

### 1.2 기존 게임에서 다루는 18개

| 묶음 | 어휘 |
|---|---|
| 생활 표현 | 여유가 있다, 평화롭다, 공기가 맑다, 따분하다, 시간 가는 줄 모르다, 불편하다 |
| 자연·농촌 명사 | 정원, 잔디, 채소, 농사, 가축, 물고기 |
| 가꾸기·동작 | 가꾸다, 깎다, 심다, 키우다, 짓다, 잡다 |

### 1.3 새 보조활동의 목표 10개

`여유가 없다`, `활기차다`, `공해가 심하다`, `편의 시설이 잘되어 있다`, `사라지다`, `생기다`, `변하다`, `몰라보다`, `상상이 되다`, `상상이 안 되다`

새 페이지의 설정 데이터는 위 10개와 기존 게임의 18개를 각각 공개한다. 자동화 검사는 두 집합의 교집합이 비어 있고, 모든 문항 정답의 합집합이 정확히 위 10개인지 확인한다.

## 2. 어휘 유형 재분류 근거

원본 어휘 카드의 단원별 범주는 목록 정리에 적합하지만, 서로 혼동하기 쉬운 의미 차이를 연습시키기에는 충분히 세밀하지 않다. 따라서 품사보다 **의미를 판단할 때 학습자가 확인해야 하는 단서**를 기준으로 네 유형으로 다시 분류했다.

| 의미 유형 | 목표 어휘 | 학습자가 확인할 핵심 단서 | 선택한 활동 |
|---|---|---|---|
| 생활환경 진단 | 여유가 없다, 활기차다, 공해가 심하다, 편의 시설이 잘되어 있다 | 시간 부족, 생기, 오염, 시설의 묶음 | 도시 단서 찾기 |
| 존재 변화 | 사라지다, 생기다 | 존재→부재, 부재→존재의 방향 | 변화 필름 맞추기 |
| 상태 변화·재인식 | 변하다, 몰라보다 | 단순한 상태 변화인지, 변화 때문에 알아보지 못했는지 | 옛사진 탐정 |
| 상상 가능성 | 상상이 되다, 상상이 안 되다 | 머릿속에 구체적인 모습이 그려지는지 | 상상 구름 고르기 |

이 재분류는 특히 `사라지다`와 `생기다`를 원본의 ‘가꾸기·동작’ 범주에서 의미 기능상 ‘존재 변화’로 옮긴다. 또한 `변하다`와 `몰라보다`를 나란히 두어 ‘변화 사실’과 ‘변화로 인한 재인식 실패’를 직접 대조한다.

## 3. 분류별 활동 설계 근거

### 3.1 도시 단서 찾기

- 대상: `여유가 없다`, `활기차다`, `공해가 심하다`, `편의 시설이 잘되어 있다`
- 구성: 8문항, 어휘마다 서로 다른 장면에서 2회 회상
- 조작: 네 단서 중 의미 판단에 꼭 필요한 두 단서를 먼저 찾고, 네 표현 중 정답을 선택

이 표현들은 한 사물의 이름이 아니라 여러 상황 단서를 종합해야 뜻이 정해진다. 예를 들어 사람이 많다는 사실만으로는 `활기차다`라고 할 수 없으며, 밝은 목소리와 움직임 같은 생기 단서가 함께 필요하다. 정답 선택 전에 핵심 단서를 찾게 하여 장면의 표면적 요소가 아니라 의미를 결정하는 특징에 주의를 모으도록 했다.

### 3.2 변화 필름 맞추기

- 대상: `사라지다`, `생기다`
- 구성: 4문항, 각 어휘 2회
- 조작: ‘먼저–나중’ 필름 사이에 변화 단어를 끌어 놓거나 눌러 배치

두 어휘의 핵심 차이는 변화의 방향이다. 정적인 뜻 풀이보다 전후 장면을 나란히 보고 단어를 그 사이에 놓게 하면 ‘있던 것→없음’과 ‘없던 것→새로 있음’을 사건 구조로 기억할 수 있다. 끌어 놓기와 누르기 방식을 모두 제공하여 스마트폰과 키보드·마우스 환경에서 같은 학습 행동을 할 수 있게 했다.

### 3.3 옛사진 탐정

- 대상: `변하다`, `몰라보다`
- 구성: 4문항, 각 어휘 2회
- 조작: 가려진 나중 사진을 연 뒤 두 장면의 관계를 판단

`변하다`는 같은 대상의 상태가 달라진 사실이 중심이고, `몰라보다`는 너무 달라져 알아보지 못한 결과가 반드시 포함된다. 모든 `몰라보다` 문항에 ‘누구인지/어디인지 알지 못했다’는 결과를 명시해 답이 장면 취향에 따라 달라지지 않도록 했다. 나중 장면을 직접 여는 행동은 전후 비교에 시선을 머물게 하는 장치다.

### 3.4 상상 구름 고르기

- 대상: `상상이 되다`, `상상이 안 되다`
- 구성: 4문항, 각 어휘 2회
- 조작: 짧은 대화를 읽고 두 생각 구름 중 하나를 선택

상상 가능 여부를 학습자의 개인 취향으로 묻지 않았다. 모든 문항에 ‘머릿속에 그려지다/떠오르다’ 또는 그 부정을 명시해 객관적으로 채점할 수 있게 했다. 긍정과 부정을 두 구름으로 항상 함께 보여 주어 한 글자 차이가 만드는 의미 대조를 반복해서 보게 했다.

## 4. 공통 학습 설계 원리

1. **인출 연습**: 설명을 먼저 읽는 대신 장면을 보고 표제어를 직접 꺼내 선택한다.
2. **의미 대조**: 혼동하기 쉬운 어휘를 같은 선택 공간에 두어 경계 특징을 비교한다.
3. **정교화**: 정답 뒤에 단순 정오 표시만 하지 않고, 왜 그 표현이 맞는지 쉬운 한국어로 알려 준다.
4. **즉시 교정과 무제한 재시도**: 오답은 실패 점수로 끝내지 않고 짧은 단서를 준 뒤 같은 문항에서 바로 고치게 한다. 첫 선택 점수는 별도로 남겨 교사가 난이도를 파악할 수 있다.
5. **반복과 장면 변화**: 각 표제어를 최소 2회, 서로 다른 장면에서 판단하게 하여 특정 문장만 외우는 것을 줄였다.
6. **시각 부호화**: 시간 부족, 오염, 전후 변화, 재인식, 생각 구름을 글자 없는 생성 삽화로 보여 주어 의미 구조와 장면을 함께 기억하게 했다.

## 5. 문항 모호성 방지 기준

- `여유가 없다`: 반드시 시간 부족이나 연속된 일정 단서를 포함한다.
- `활기차다`: 사람 수뿐 아니라 밝은 소리·움직임·생기 단서를 포함한다.
- `공해가 심하다`: 매연, 오염된 물, 나쁜 냄새처럼 환경 오염 단서를 포함한다.
- `편의 시설이 잘되어 있다`: 병원, 가게, 교통 등 두 종류 이상의 생활 시설을 포함한다.
- `사라지다`와 `생기다`: 전후 장면에서 존재 방향이 하나로 결정되게 한다.
- `변하다`: 같은 대상임을 알아보는 상황으로 제한한다.
- `몰라보다`: 알아보지 못한 결과를 문장에 명시한다.
- 상상 표현: ‘그려지다/떠오르다’의 긍정 또는 부정을 문장에 명시한다.

## 6. 화면·접근성·저장 설계

### 6.1 학습 우선 화면

- 긴 사용 설명 없이 제목, 활동 탭, 첫 장면과 첫 조작을 바로 배치했다.
- 모바일 360×740에서도 첫 단서 버튼이 첫 화면 안에 나타난다.
- 활동 이동은 네 개의 짧은 탭으로 제공하고, 각 탭에 완료 수와 완료 표시를 보여 준다.
- 학습자에게 보이는 제목, 버튼, 피드백, 대체 설명은 한국어로 작성했다.
- 기존 어휘 그림에는 영어 문자가 남은 경우가 있어 사용하지 않았다. 새 활동 전용 삽화를 글자 없이 생성하고 한국어 장면 설명을 함께 제공했다.

### 6.2 반응형·접근성

- 스마트폰에서는 활동 탭을 가로 스크롤로 바꾸고 장면과 문제를 세로로 배치한다.
- 태블릿과 PC에서는 장면과 판단 영역을 나란히 보여 준다.
- 주요 조작은 최소 44px 이상의 터치 영역을 갖는다.
- 모든 조작은 기본 `button` 요소로 제공하며, 탭은 좌우 방향키로도 이동할 수 있다.
- 활동 탭은 한 번에 하나만 탭 순서에 들어가는 방식으로 구성하고, 활동 패널과 이름을 연결했다.
- 단서·정답·사진 공개 뒤 화면을 다시 그려도 초점이 다음 학습 행동으로 이어진다.
- 끌어 놓기 활동에는 누르기 대체 조작이 있다.
- 상태 변화는 글자, 색, 기호를 함께 써서 색만으로 정오를 알리지 않는다.
- `prefers-reduced-motion`에서는 이동·흔들림 효과를 사실상 제거한다.

### 6.3 자동 저장

다음 상태를 답이나 조작이 일어날 때마다 `localStorage`에 저장한다.

- 현재 학습소와 현재 문항
- 찾은 핵심 단서
- 열어 본 나중 사진
- 오답을 포함한 선택 기록
- 첫 선택 정답 수와 문항 완료 상태

새로고침 뒤에는 같은 학습소와 문항, 정답 피드백까지 복원한다. 저장소를 사용할 수 없는 환경에서도 활동은 계속 작동한다. 전체 초기화는 별도 버튼을 누른 뒤 확인해야 실행된다.

### 6.4 생성 삽화와 경량화

- 기본 이미지 생성 도구로 2×2 원본 시트 5장을 제작했다.
- 프롬프트 묶음은 ‘도시 생활 핵심 4장’, ‘도시 생활 추가 4장’, ‘존재 변화 전후 4쌍’, ‘상태 변화·재인식 전후 4쌍’, ‘상상 가능성 4장’이다.
- 모든 프롬프트에 글자, 숫자, 로고, 상표, 워터마크를 넣지 않도록 명시했다.
- 전후 변화는 한 사분면 안에 두 시점을 함께 생성해 카메라 위치와 대상의 일관성을 유지했다.
- 원본 시트는 `backup/asset-sources/c14/vocab-support-lab/source-sheets/`에 보관하고, 실제 페이지에는 각 장면만 잘라 쓴다.
- 최종 에셋은 512×384 RGB WebP, 품질 76, 메타데이터 제거 방식으로 만들었다.
- 최종 20장 합계는 519,604바이트이고, 가장 큰 파일도 39,884바이트로 64KiB 기준보다 작다.
- 현재 문항의 이미지 한 장만 `eager`로 불러오고, 다른 19장은 해당 문항에 도착하기 전까지 요청하지 않는다.
- 이미지는 장식 중복 낭독을 막기 위해 빈 대체텍스트와 `aria-hidden="true"`를 사용하며, 바로 옆 한국어 장면 설명이 의미를 전달한다.

  - 단서·정답·사진 공개 뒤 키보드 초점 유지와 탭 패널 연결

저장소 전체 정적 정책 검사는 다른 과·앱에 이미 존재하던 외부 글꼴과 이미지 속성 문제 때문에 일부 실패했지만, 실패 목록에 이번 새 파일과 수정 파일은 포함되지 않았다.

## 9. 기대 학습 결과

학습자는 10개 표현을 단순히 뜻과 짝짓는 데서 끝나지 않고, 생활 장면의 핵심 단서를 찾고, 변화 방향을 구성하고, 변화의 결과를 구별하고, 상상 가능 여부를 판정한다. 따라서 실제 말하기·읽기에서 표현을 선택할 때 필요한 의미 기준을 장면과 함께 기억할 수 있다.
