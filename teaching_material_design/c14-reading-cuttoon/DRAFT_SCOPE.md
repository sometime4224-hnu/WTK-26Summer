# c14 읽기 컷툰 설계 검토 초안

상태: `C14RC-VISUAL-009` 고증 개선 무문자 이미지 재생성·연결 및 기술 검증 완료
지원 범위: `LIMITED` 읽기 보조
구현 기준: c13의 문장-컷 동기화 패턴 + c14의 기존 본문/컷 쓰기 자료

## 제안하는 최소 구현

- 정확한 목표 표현 후보: `도낏자루 썩는 줄 모른다`
- 한국어 전용으로 구현하고, 검토되지 않은 다국어 뜻·예문은 넣지 않는다.
- c13의 이전/다음, 썸네일, 문장 선택, 좌우 키, 모바일 접기, 이미지 실패 대체 흐름을 사용한다.
- 새 점수 문항은 만들지 않는다. 컷 이동은 탐색형 활동으로만 둔다.
- 기존 `c14/writing-cut.js`의 12개 문장과 장면 대체 텍스트를 정확히 재사용한다. 이번 읽기 컷툰에는 핵심어 칩을 넣지 않는다.
- 닫힌 선택 도움말은 먼저 노출하지 않고, 버튼 이름은 `이동 도움말`로 분리한다.
- 조선 후기 평민의 상투·망건·저고리·바지·대님·짚신과 농촌 한옥을 명시해 새로 생성한 무문자 2열×3행 WebP 스프라이트 2개로 12장면을 표시한다. 실제 셀 비율 2:1을 그대로 사용해 왜곡을 막고, 이미지 로드 실패 때에는 같은 의미의 CODE 대체 장면을 제공한다.
- 기본 보기에서는 그림을 왼쪽, 읽기 지문을 오른쪽에 둔다. 큰 그림 보기에서는 읽기 지문 목록을 접고 현재 문장을 큰 그림 위에 표시하며 이전/다음 버튼과 좌우 방향키로 이동한다.
- `reading_cuttoon_c14_modern_v1`에 현재 컷, 모바일 패널 접기 상태와 보기 모드를 저장한다.

## 12단계 장면 매핑

| 컷 | 기존 장면 대체 텍스트 | 기존 문장 범위 |
|---:|---|---|
| 1 | 깊은 산 동굴 앞에 선 나무꾼 | 동굴을 발견했다. |
| 2 | 작은 동굴 안쪽이 넓고 밝아지는 장면 | 점점 넓어지고 밝아졌다. |
| 3 | 동굴 안에서 수염이 긴 노인들이 바둑을 두는 장면 | 바둑을 두고 있었다. |
| 4 | 바둑이 재미있어서 시간 가는 줄 모르고 구경하는 나무꾼 | 시간 가는 줄 모르고 구경했다. |
| 5 | 집에 돌아가려다 도끼를 보고 놀라는 나무꾼 | 도끼를 들다가 깜짝 놀랐다. |
| 6 | 자루가 없는 도끼를 들고 산을 내려오는 나무꾼 | 자루가 없는 도끼를 들고 내려왔다. |
| 7 | 완전히 달라진 마을에 도착한 나무꾼 | 집과 사람들이 달라졌다. |
| 8 | 낡은 집과 지붕을 덮은 큰 나무를 보는 나무꾼 | 큰 나무가 지붕을 덮고 있었다. |
| 9 | 지나가는 사람에게 자기를 아느냐고 묻는 나무꾼 | 자기 이름을 말하며 물어보았다. |
| 10 | 지나가던 사람이 나무꾼에게 할아버지의 할아버지 이야기를 하는 장면 | 못 돌아온 조상이라고 대답했다. |
| 11 | 나무꾼이 봤던 노인들이 신선들이었다는 장면 | 노인들은 신선들이었다. |
| 12 | 신선들의 바둑을 구경하는 동안 100년이 지난 장면 | 100년의 시간이 지났다. |

실제 학습자 화면에는 위 표의 요약문을 새 본문으로 쓰지 않고, `c14/writing-cut.js`의 정확한 문장만 표시한다.

## 승인된 교사 결정

1. 목표 표현은 `도낏자루 썩는 줄 모른다`로 확정한다.
2. 컷툰은 “본문 문제 풀이 뒤 복습”으로 제한한다.
3. 본문 1번 정답인 `도낏자루가 다 썩어서 없어졌기 때문이다.`는 컷 문장에 넣지 않고 기존 12개 문장만 사용한다.
4. 기존 12개 WebP 대신 정책을 충족하는 6컷 AI 생성 WebP 스프라이트 2개를 사용하고, 로드 실패 때에만 CODE 대체 장면을 사용한다. 정상 UI의 인라인 SVG는 0개로 유지한다.
5. 로컬 기존 본문과 문장 재사용 범위를 승인한다.
6. 기본 보기의 그림을 왼쪽으로 옮기고, `큰 그림 보기`/`기본 보기` 전환과 문장 위·큰 그림 아래 레이아웃을 추가한다. 두 모드 모두 이전/다음 이동과 기존 문장 동기화를 유지한다.
7. 사용자 제공 ChatGPT Image의 인물·머리 모양을 채택하고 이미지 안 글자 없음 조건은 적용하지 않는다. 다만 승인된 본문과 직접 충돌하는 6번 컷 문구는 크롭에서 제외한다.

승인 근거: 1~5는 2026-07-15 교사의 “모두 승인”, 6은 같은 날 교사의 직접 레이아웃 변경 요청과 비활성화 요청 폐기 후 작업 재개 지시다.

- `images/writing-cut`: 12개, 896,866바이트. 2개는 파일당 80KiB를 넘고 전체 개수·용량도 한도를 넘는다.
